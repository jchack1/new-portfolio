Example jekyll post

December 11, 2019

Update - I got the theme to work! In our assignment guidelines we were told to choose from one of the github pages themes. I found that I didn't particularly like most of the themes, especially as a design person. They were just not my style. I chose the "minimal" theme and I decided I would just try to customize it more to my liking.

I struggled in class this week to get a theme to work. I suppose that is just because I didn't know what I was doing yet. I must have been missing some critical yet small piece to the puzzle. Today I tried this strategy and had success:

  • set up my site folder with a few pages, posts, layouts, navigation, and data
  • download the theme folder from github onto my machine
  • copied files one by one into my site folder, being careful about which ones I wanted to replace
  • made sure the gem file and config file were from the minimal theme
  • used command "bundle install"
  • then command "bundle exec jekyll serve"
  • realized that I needed the minimal layouts in my site folder, made a few changes to my pages
  • now I am seeing my pages, with the minimal theme. Yay!

First post

December 9, 2019

Hello, there! As you can tell from the title, this is the first post on my portfolio blog. Ideally I would like to write blog posts as I continue learning web development.

We'll have to see if I actually keep this up. I blogged while creating my jekyll site as I was building it, and I found it was a fun process. I enjoyed writing about the successes and failures I was experiencing while building my site, and I actually went back to read previous posts to see my process and remember some of the things I tried. I would like to do this while I continue to learn, because I know I have so much left to learn. Perhaps my posts may be useful to someone else in the future. If nothing else, hopefully they will allow me to see how far I have come since starting in this field.

I feel as though a static site like I currently have is not the best way to deal with blog posts. I am debating creating a back-end with node.js. This may work better for blogs as I can use javascript on ejs pages, or something similar. Also, having to go in and update every single nav or footer whenever I make a change is a pain. So I think making a back-end will be important.

Anyway, that's it. Right now I think I will fill this up with a few more posts from my jekyll site, just to add some content.
